# ElderScrollsIdle — SvelteKit Edition
> Godville-like idle RPG in the world of The Elder Scrolls

## Анализ старого проекта (bg-godv / Next.js)

### Источник
- Репозиторий: https://github.com/thejuzvr/bg-godv
- Стек: Next.js + PostgreSQL + Redis + BullMQ + Socket.IO + Drizzle ORM + shadcn/ui
- Проблемы оригинала: громоздкая архитектура (3 отдельных процесса), невалидные компоненты, сложная маршрутизация Next.js API routes

## Проблемы оригинального проекта

1. **3 отдельных процесса**: `npm run dev` + `npm run worker` + `npm run realtime` — сложная синхронизация
2. **Громоздкость Next.js**: App Router, server actions, middleware — оверхед для игры
3. **Socket.IO поверх Redis**: сложная цепочка (Socket.IO → Redis pub/sub → clients)
4. **Невалидные компоненты**: ошибки во фронтенде React

## Новый проект: SvelteKit TES Idle

### Стек
| Слой | Технология | Причина |
|------|-----------|--------|
| Frontend | SvelteKit 5 | Svelte 5 Runes ($state, $derived) для реактивности |  
| State | gameStore.svelte.ts | Синглтон на рунах для глобального состояния |
| Backend | SvelteKit + ws | Единый процесс, нативные вебсокеты |
| Auth | Better-Auth | Современная и безопасная аутентификация |
| DB | PostgreSQL + Drizzle | Мощная типизация и производительность |
| Cache/Queue | Redis + BullMQ | Надежная очередь тиков персонажа |
| UI | Vanilla CSS + Custom Tokens | Полный контроль над дизайном (Gold: #d4af37) |
| Icons | Lucide Svelte | Стандарт индустрии |

### Ключевые принципы нового проекта

#### 1. Монолит вместо 3 процессов
SvelteKit's `hooks.server.ts` запускает background worker при старте — один процесс управляет всем.

#### 2. Native WebSocket + Vite Integration
Интеграция через `vite.config.ts` в режиме разработки и `handleUpgrade` в продакшене. Менеджер `wsManager` управляет подключениями и рассылкой событий.

#### 3. Real-time State Management (Svelte 5)
Вместо множества сторов используется один `gameStore.svelte.ts`, построенный на `$state`. Это обеспечивает мгновенную реактивность UI при получении событий через сокет или при ручном обновлении.

#### 4. Offline Processing & Sync
При входе игрока SvelteKit загружает последние события и состояние персонажа, инициализируя магазин. Фоновые тики продолжают идти в BullMQ воркере.

### Игровой цикл (новый)

```
hooks.server.ts (startup)
    └─► startGameWorker()
            └─► BullMQ Queue: "game-tick"
                    └─► processCharacter(character)
                            └─► AI Brain: brain.ts
                                    └─► Game Engine: game-engine.ts
                                            ├─► Update character state
                                            ├─► Save to PostgreSQL
                                            └─► Publish to Redis
                                                    └─► WebSocket broadcast
                                                            └─► Client UI update
```

## Важные технические заметки

### WebSocket в SvelteKit
SvelteKit не имеет встроенного WebSocket, поэтому используем кастомный сервер или `vite-plugin-svelte-kit-ws`.

Вариант: через `hooks.server.ts` + `@fastify/websocket` или через Vite Development proxy.
Для прода: SvelteKit Node adapter поддерживает `handleUpgrade`.

### Offline Mode Logic
```typescript
// При каждом тике
await db.update(characters)
  .set({ lastProcessedAt: Date.now() })
  .where(eq(characters.id, characterId));

// При входе игрока
const offline = Date.now() - character.lastProcessedAt;
const ticksMissed = Math.floor(offline / ADVENTURE_TICK_INTERVAL);
const scaledTicks = Math.min(ticksMissed, MAX_OFFLINE_TICKS); 
// Запускаем accelerated offline processing
```

### BullMQ вместо setInterval
- Персонаж добавляется в очередь при создании
- Каждый воркер обрабатывает одного персонажа
- Delay после каждого тика (15-40с или 3-5с)
- Concurrency настраивается
